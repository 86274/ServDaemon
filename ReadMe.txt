Клиент серверное приложение под Linux

Алгоритм приложения.
1. Запускается сервер. Сервер ожидает подключение клиента. Сервер пишет в логи сообщение о своём запуске. 
2. Запускается клиент с одним параметром, которым указывается название файла. Клиент открывает файл.
Пример: 
	./Client text.txt
Если сервер не запущен, то клиент будет ожидать связи с сервером.
3. Устанавливается связь. 
Клиент принимает сообщение от сервера и сообщает, что подключение установлено.
4. Клиент считывает данные с файла и отправляет на сервер.
5. Сервер открывает/создаёт файл, принимает данные, записывает их в файл.
6. Сервер отправляет сообщение клиенту о выполненной операции.
7. После завершения передачи файла Сервер переходит в режим прослушивания, а Клиент завершает работу.
8. По сигналам SIGTERM, SIGHUP сервер корректно завершается - о чём сообщает записью в логи.
